## Core Narrative (use this in all post writing)
BBT is not a course platform. The model is:
1. Student enrolls in a **Track** (not a course)
2. Goes through **mentorship + shadowing** — learning by doing on real projects with senior engineers
3. Exits one of two ways:
   - Gets **absorbed into BBT's own software house** (direct employment)
   - Is so skilled from the process that they can **go anywhere** in the industry
> The message: "We don't just teach you. We build you into someone the industry actually wants."

**Messaging angles to use:**
- "We don't just graduate students — we absorb the best into our own team."
- "Train under real engineers. Ship real software. Get hired — or join us."
- "Our tracks don't end with a certificate. They end with a career."
- "Shadow. Build. Ship. Repeat — until you're unstoppable."

## The Honest Industry Critique (BBT's founding belief)
The skills training industry is bound to fail — because selling courses without a system to make students employable is a broken promise. You can teach all you want, but if there's no pipeline, no accountability, no real-world integration — the outcome is just another certificate collecting dust.

**BBT's position:** We refuse to offer courses we can't stand behind. Unless there is a deliberate system in place — mentorship, shadowing, real projects, direct employment pathways — we cannot and will not guarantee employability. That system *is* BBT's Tracks model.

> "Anyone can teach skills. Very few can guarantee what happens after."

**This is why BBT built a software house alongside the academy** — not as a separate business, but as the proof of the model. When our graduates get absorbed into our own teams, that's not a marketing claim. That's accountability.

**Use this angle when:**
- Critiquing the broader "skills bootcamp" industry
- Differentiating BBT from competitors
- Speaking to skeptical students who've been burned by other institutes
- Addressing employers who doubt training institute graduates

## Typography Rules
- **Headlines:** BOLD, ALL CAPS, large
- **Two-tone headline style:** First line in WHITE, key/emphasis word in ORANGE
  - Example: "FROM VISION TO / **SCALABLE** / SOFTWARE"
  - Example: "TECH TALENT / **IN PAKISTAN**"
- **Body copy:** Regular weight, white, concise (2–3 lines max per slide)
- **Tags/Pills:** Rounded outlined pills with white text — used for topics (e.g., "Machine Learning", "EdTech", "Full Stack Dev")
- **Stats:** Orange number, white label underneath, inside rounded-corner cards

## Visual Design Rules
- **Background texture:** Subtle binary code (0s and 1s) pattern on dark background — always present
- **Decorative element:** Large faded slide number (01, 02, 03...) in top-right as watermark
- **Wavy lines:** Thin orange/gold curved lines on corners/edges for visual depth
- **Icons:** Emojis (🧠🚀💡🌐) used as section icons on carousel slides — keeps it human and approachable
- **Stats cards:** 2–4 metric cards in a grid, orange value + white label
- **Slide counter:** Bottom-left in small text (01 / 06 format)
- **Globe / tech imagery:** Used for international/software themes — dark with orange glow

## Website Development Rules (bbt.edu.pk)

### Stylesheet — bbt.css
- **All styles live in `bbt.css` only.** Never write `<style>` blocks inside HTML files.
- Every HTML page must link the stylesheet in `<head>`:
  ```html
  <link rel="stylesheet" href="bbt.css">
  ```
- When adding new CSS for a new page or component, append it to `bbt.css` with a clearly labelled section comment.

### New Pages — use template.html
- **Always copy `template.html` as the starting point** for any new page.
- `template.html` includes: nav, page-hero placeholder, example section, CTA/apply section, footer, theme toggle script, and scroll-reveal script.
- Set `class="active"` on the correct nav link for the current page.
- Replace `PAGE TITLE`, `PAGE DESCRIPTION`, `PAGE-SLUG` in `<head>` with real values.

### Required `<head>` block for every page
```html
<!-- Fonts -->
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Barlow:ital,wght@0,300;0,400;0,500;0,600;0,700;0,800;0,900;1,400;1,700&family=Barlow+Condensed:wght@600;700;800;900&family=DM+Mono:wght@400;500&display=swap" rel="stylesheet">

<!-- BBT Design System -->
<link rel="stylesheet" href="bbt.css">

<!-- Theme: must be before </head> to prevent flash -->
<script>(function(){var t=localStorage.getItem('bbt-theme')||'dark';document.documentElement.setAttribute('data-theme',t)})()</script>
```

### Required scripts for every page (before `</body>`)
```html
<script>
function toggleTheme() {
  var html = document.documentElement;
  var next = html.getAttribute('data-theme') === 'light' ? 'dark' : 'light';
  html.setAttribute('data-theme', next);
  localStorage.setItem('bbt-theme', next);
  var btn = document.getElementById('theme-toggle');
  if (btn) btn.textContent = next === 'light' ? '☀️' : '🌙';
}
window.addEventListener('DOMContentLoaded', function() {
  var t = localStorage.getItem('bbt-theme') || 'dark';
  var btn = document.getElementById('theme-toggle');
  if (btn) btn.textContent = t === 'light' ? '☀️' : '🌙';
  document.body.classList.add('js-ready');
  var io = new IntersectionObserver(function(entries) {
    entries.forEach(function(e) { if (e.isIntersecting) e.target.classList.add('in'); });
  }, { threshold: 0.12 });
  document.querySelectorAll('.reveal').forEach(function(el) { io.observe(el); });
});
</script>
```
